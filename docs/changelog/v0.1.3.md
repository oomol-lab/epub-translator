# Release v0.1.3

This release focuses on improving EPUB compatibility and optimizing the translation pipeline.

## üêõ Bug Fixes

### EPUB Format Compatibility (#96)

- Enhanced media type recognition in spine processing
- Added self-closing tag normalization for 21 standard HTML5 void elements
- Improved compatibility with non-standard EPUB formats

Related PR: https://github.com/oomol-lab/epub-translator/pull/96

### Mimetype File Ordering (#98)

Fixed mimetype file position to appear as the first entry in ZIP files per EPUB OCF specification.

Related PR: https://github.com/oomol-lab/epub-translator/pull/98

### Stream Processing Optimization (#101)

Refactored XMLStreamMapper from batch processing to single-element processing, simplifying internal logic.

Related PR: https://github.com/oomol-lab/epub-translator/pull/101

## ‚ú® Features

### Enhanced Fill Template Algorithm (#99)

Improved fill.jinja template with semantic type matching instead of position-based matching. Added formal algorithm steps and special-case rules for better error recovery.

Related PR: https://github.com/oomol-lab/epub-translator/pull/99

### Cache Seed Content Support (#100)

Added `cache_seed_content` parameter for smarter cache invalidation. Runtime version and target language are now included in cache keys, enabling automatic cache invalidation on version updates.

Related PR: https://github.com/oomol-lab/epub-translator/pull/100

### Fragment Merging Refactor (#102)

Restructured stream processing logic with serial group processing. Better handling of nested content in resource segments.

Related PR: https://github.com/oomol-lab/epub-translator/pull/102
